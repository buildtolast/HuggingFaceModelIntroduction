1. How Models Work: Transformer Architecture

TL;DR: Modern language models use the Transformer architecture, which processes text through layers of attention mechanisms. Each layer attends to relationships between tokens (words/subwords), allowing the model to understand context and generate coherent text.

The Transformer Block

Every modern language model (GPT, Llama, Mistral, Qwen) is built from stacked Transformer blocks. Here's the structure:

INPUT TOKENS (text tokenized into IDs) ↓ EMBEDDING LAYER ├─ Token Embedding: Maps token IDs to 768/1024/etc dimensional vectors └─ Position Encoding (RoPE): Encodes position information ↓ [TRANSFORMER BLOCK] × N (e.g., 32 blocks for 7B model) ├─ Layer Norm ├─ Multi-Head Self-Attention (8/12/32 heads) │ ├─ Query (Q): "What am I looking for?" │ ├─ Key (K): "What information do I have?" │ ├─ Value (V): "What should I output?" │ └─ Softmax(Q·K/√d)·V: Compute attention weights ├─ Residual Connection + Layer Norm ├─ Feed-Forward Network (2 linear layers + activation) └─ Residual Connection ↓ OUTPUT LOGITS (one probability distribution per token) ↓ SAMPLING / DECODING └─ Choose next token based on probabilities

Key Concepts Explained

🔹 Tokens & Tokenization

Text is split into tokens (usually subwords). Example:

Input: "Hello, world!"
Tokens: ["Hello", ",", " world", "!"]
Token IDs: [1523, 45, 896, 328]
Context window: Max 128K tokens (roughly 96K words)

Why it matters: Token count directly affects memory usage (O(n²) complexity in attention). 128K context = 128K × 128K attention matrix = expensive!

🔹 Attention Mechanism (Query-Key-Value)

Self-Attention Formula:

Attention(Q, K, V) = Softmax(Q·K^T / √d_k)·V Where: - Q (Query): What am I interested in? (N×d) - K (Key): What are my features? (N×d) - V (Value): What should I output? (N×d) - d_k: Dimension of keys (usually 64 for 8-head attention on 512-dim model) - N: Sequence length (context window size)

Multi-Head Attention: Instead of one 512×512 attention, use 8 heads of 64×64. Allows model to attend to different aspects simultaneously.

🔹 Position Encoding (RoPE)

Modern models use Rotary Position Embeddings (RoPE) instead of absolute position encoding.

RoPE applies a rotation matrix to Q and K based on position: - Position 0: rotate 0° - Position 1: rotate θ° - Position 2: rotate 2θ° - ... Benefits: ✓ Extrapolates to longer sequences (important for extending context) ✓ Works naturally with attention mechanism ✓ Better than absolute positional embeddings

2. Transformer Paradigms

There are three main transformer architectures. In 2026, decoder-only models dominate, but understanding all three is important:

Encoder-Only (BERT)

Architecture: Bidirectional attention

  • ✓ Sees left AND right context
  • ✓ Great for understanding/classification
  • ✗ Cannot generate text

Examples: BERT, RoBERTa, DeBERTa

Best For: Text classification, NER, embeddings

Decoder-Only (GPT)

Architecture: Causal attention (only left context)

  • ✓ Can generate text
  • ✓ Versatile for any task
  • ✓ Scales well to large sizes
  • ✗ Slower for classification tasks

Examples: GPT, Llama, Mistral, Qwen

Best For: Everything (chat, code, reasoning)

Encoder-Decoder (T5)

Architecture: Two stacks

  • ✓ Encoder bidirectional, Decoder causal
  • ✓ Good for seq2seq (translation, summarization)
  • ✗ Requires more compute
  • ✗ Less versatile than decoder-only

Examples: T5, BART, mBART

Best For: Translation, summarization

Why Decoder-Only Won: GPT-3 showed that scaling decoder-only models on unsupervised text data leads to emergent abilities (few-shot learning, reasoning). Instruction-tuning converts them into general-purpose assistants. This unified approach beats specialized models.

3. Autoregressive Generation & Decoding

Models don't output full text at once. They generate one token at a time, conditioning each new token on all previous tokens. Here's how:

Two-Phase Inference

PHASE 1: PREFILL (Process your prompt) ───────────────────────────────────── Input: "What is machine learning?" Process all tokens through model in parallel Output: Logits for next token position Time: ~1 second (depends on prompt length) → Compute KV-cache for all positions PHASE 2: DECODE (Generate output tokens) ────────────────────────────────────────── Iteration 1: Input: [What, is, machine, learning?, ] Output: Logits → Sample next token Iteration 2: Input: [What, is, machine, learning?, , ] Output: Logits → Sample next token (repeat until max_tokens or stop token) Time: ~100ms per token (sequential)

Token Sampling Strategies

Given logits (raw scores) for 50,000 vocabulary words, how do you pick the next token?

1. Greedy Decoding

next_token = argmax(logits)

Behavior: Always picks highest probability

Pros: Deterministic, fast

Cons: Repetitive, boring

Use: When you need reproducibility

2. Temperature Sampling

p = Softmax(logits / temperature) next_token = sample(p) T=0.1 → Sharp (deterministic) T=1.0 → Normal (balanced) T=2.0 → Flat (very random)

Pros: Simple, controls randomness

Cons: Can sample low-prob tokens

Use: Basic control over creativity

3. Top-K Sampling

# Keep only top K tokens by probability top_k = 50 filtered = filter_top_k(logits, k=50) next_token = sample(filtered)

Pros: Prevents very unlikely tokens

Cons: Fixed K doesn't adapt

Use: Older standard (2020-2022)

4. Top-P (Nucleus) ⭐ BEST

# Cumulative probability = P # Include tokens until sum ≥ P top_p = 0.9 candidates = cumsum(sorted(logits)) filtered = candidates[cumsum <= 0.9] next_token = sample(filtered)

Pros: Adaptive, removes low-prob tail

Cons: Slight overhead

Use: Industry standard (recommended)

Nucleus Sampling Visualization

Example: On confident tokens (high entropy), nucleus 0.9 might select 3 tokens. On uncertain tokens (low entropy), it adapts to 50+ tokens. This prevents "overthinking" on easy tokens and allows exploration on hard decisions.

Critical: These parameters interact with each other. Changing one affects the optimal values of others. Understanding tradeoffs is essential.

Context Window

What it is

Maximum number of tokens the model can process in one request. Examples:

  • 4K context = ~3,000 words
  • 128K context = ~96,000 words
  • 10M context = ~7.5M words (2026 models)

Known Issues

  • Lost in the Middle: Models perform worse on information in the middle of long contexts
  • Positional Extrapolation: Models trained on 4K context may hallucinate at 32K (unless using RoPE interpolation)
  • KV-Cache Linearity: During decode phase, KV-cache grows linearly, limiting batch size

When to Use What

RAG vs Stuffing

Stuffing: Put everything in context

System: You are helpful. Context: [100KB of documents] User: What is the main topic? → 128K context needed

RAG (Retrieval-Augmented Generation): Retrieve only relevant chunks

User: What is the main topic? → Search: Find 3 relevant documents System: You are helpful. Context: [3 relevant chunks, ~5KB] → 8K context needed (much faster!)

Recommendation: For production, always use RAG. It's faster, cheaper, and often more accurate.

Temperature

What it is

Controls randomness of sampling. Divides logits before computing probabilities:

p = Softmax(logits / temperature) Temperature values: - 0.0 = Deterministic (always same output) - 0.7 = Default (balanced) - 1.0 = Normal softmax (base behavior) - 1.5 = More creative - 2.0 = Very random

Top-P (Nucleus Sampling)

What it is

Filters tokens to only those whose cumulative probability ≥ top_p. Combines with temperature.

# Standard values top_p = 0.9 → Include until cumulative prob = 90% # Behavior on different token distributions: Confident tokens (e.g., "the", "is"): Probabilities: [0.70, 0.15, 0.10, 0.05] Cumulative: [0.70, 0.85, 0.95, 1.0] top_p=0.9 selects: first 3 tokens Uncertain tokens (e.g., creative writing): Probabilities: [0.05, 0.05, 0.05, ..., 0.05] (many equal) top_p=0.9 selects: many tokens (~18 tokens)

Top-P vs Top-K Comparison

Aspect Top-K (top_k=50) Top-P (top_p=0.9)
Adapts to confidence? ✗ No (always 50 tokens) ✓ Yes (adaptive)
Confident prediction Force considers 50, wastes compute Considers 2-3 tokens (efficient)
Uncertain prediction Limited to top 50 (may cut off good tokens) Explores all until cumulative 0.9
Recommendation Deprecated (2020-2022) ⭐ Industry standard

Tuning Top-P

  • 0.9: Default (remove bottom 10% probability tail)
  • 0.95: More permissive (include more tokens)
  • 0.5: More restrictive (fewer choices)
  • 1.0: Disables nucleus sampling (use full distribution)

Presence vs Frequency Penalties

Presence Penalty

Binary: penalize any token that appeared before (regardless of count)

presence_penalty = 0.6 # Range: -2.0 to 2.0 If token appeared before: logit -= presence_penalty Effect: Encourages diversity, penalizes repetition

Use: When you want the model to introduce new topics

Frequency Penalty

Graduated: penalize tokens proportional to how often they appeared

frequency_penalty = 0.6 # Range: -2.0 to 2.0 logit -= frequency_penalty * count(token) Effect: Proportional penalty to frequency Example: Token appeared 1x → logit -= 0.6 Token appeared 3x → logit -= 1.8

Use: Standard for avoiding repetition (more sophisticated)

When to Use Each

Aspect Presence Frequency
Mechanism Binary (appeared or not) Graduated (proportional)
Subtlety Heavy-handed More nuanced
Best For Avoiding exact repeats General conversation
Default Value 0.0 (disabled) 0.0 (disabled)
Good Values 0.5-1.0 0.1-0.3

Other Important Parameters

Max Tokens

Maximum number of tokens to generate. Must be ≤ (context_window - prompt_length)

max_tokens = 200 Total tokens = prompt_length + max_tokens ≤ context_window

Stop Sequences

Stop generation when these tokens appear:

stop = ["\n\n", "User:", "==="] Useful for: - Preventing model from continuing user turn - Stopping at section breaks - Custom formatting

Repetition Penalty

Penalize any appearance of a token (different from frequency penalty):

repetition_penalty = 1.1 # Range: 1.0 to 2.0 1.0 = No penalty 1.2 = Moderate penalty 1.5 = Strong penalty

Parameter Interaction Examples

Example 1: Creative Writing
temperature = 1.2 # More randomness top_p = 0.95 # Allow more tokens presence_penalty = 0.1 # Slight diversity push frequency_penalty = 0.2 Result: Varied, creative text with some redundancy
Example 2: Code Generation
temperature = 0.2 # Deterministic top_p = 0.9 # Standard presence_penalty = 0.0 frequency_penalty = 0.0 max_tokens = 500 stop = ["\ndef ", "\nclass "] Result: Consistent, focused code generation

Quantization Methods

FP32 (Full Precision)

Bits per weight: 32 Size (7B): 28GB Quality: 100% Speed: Baseline VRAM: High

Use: Training, research where quality is paramount

FP16 (Half Precision)

Bits per weight: 16 Size (7B): 14GB Quality: 99.5% Speed: 2× faster VRAM: 50% savings

Use: Standard for most inference (GPU default)

INT8 (8-bit Integer)

Bits per weight: 8 Size (7B): 7GB Quality: 98-99% Speed: 2-3× faster VRAM: 75% savings

Use: Aggressive reduction with minimal quality loss

GPTQ (4-bit)

Bits per weight: 4 Size (7B): 3.5GB Quality: 95-96% Speed: 2.6× faster VRAM: 87% savings

Use: Mobile/edge when speed matters most

AWQ (4-bit Adaptive)

Bits per weight: 4 (adaptive precision) Size (7B): 3.5GB Quality: 96-97% (better than GPTQ) Speed: 10.9× faster (Marlin kernel) VRAM: 87% savings

Use: Best 4-bit method (2024+)

GGUF (Portable)

Bits per weight: 4-8 (configurable) Size (7B): 3.5-7GB Quality: 96-99% Speed: Universal VRAM: CPU-friendly Tools: Ollama, llama.cpp

Use: Cross-platform deployment (CPU + GPU)

6. Model Evolution Timeline (2017-2026)

2017
The Transformer Era Begins
Attention Is All You Need (Vaswani et al.) introduces the Transformer architecture. Foundation of all modern LLMs. Parallel attention mechanisms replace RNNs.
2018-2019
BERT & GPT-1/2 Era
BERT (Google): Bidirectional transformer, dominates NLP tasks. GPT-1/2 (OpenAI): Decoder-only, shows promise of scaling. Still modest by 2026 standards (100M-1.5B params).
2020
Scaling Laws Discovered
GPT-3 (175B parameters): OpenAI releases GPT-3. Demonstrates "scaling laws" — performance improves predictably with more parameters. Few-shot learning emerges. Costs $4.6M to train.
2021-2022
Open-Source Awakening
T5, BLOOM, EleutherAI models: Open-source alternatives emerge. InstructGPT (GPT-3.5): RLHF (instruction-following via human feedback) becomes standard. Paves way for ChatGPT.
Nov 2022
ChatGPT Explosion
ChatGPT launches: 1M users in 5 days, 100M in 2 months. Accessible, user-friendly chat interface changes everything. Opens multi-billion dollar AI market.
2023
Open-Source Flourishes
Llama (Meta): 7B-65B models leaked, become most-used base models. Mistral, Qwen, Yi: Fast-moving startups ship competitive models. Claude (Anthropic): Constitutional AI approach differs from pure scaling. Open-source community becomes viable.
2024
Multimodal & Efficiency Turn
GPT-4V, Llama Vision, Qwen-VL: Vision-language models mainstream. Mixture of Experts (MoE): Efficient scaling via Mixtral, Qwen3. Chinese models surge (Qwen, DeepSeek). Flash Attention 2 reduces inference cost. China overtakes US in model downloads (55% vs 30%).
2025
Reasoning & Context Explosion
o1, DeepSeek-R1: Reasoning models with chain-of-thought scaling. Context windows explode: 1M tokens (Gemini), 10M tokens (Grok-2). Multimodal dominates: Most new models support images. Quantization standard: AWQ, GGUF become mature. Small models still win adoption (7-13B sweet spot remains for production).
2026 (Now)
Specialization & Efficiency Era
Model Landscape: 2.7M models on Hugging Face. Chinese dominance confirmed: Qwen in 8 of top 10 most-downloaded models. Size paradox: While 1T+ models exist, 7-13B models dominate actual deployment. MoE standard: Mixture of Experts now common for larger models. RAG ubiquitous: Long context used more for RAG than stuffing. Edge computing surge: On-device inference (Gemma 2B, Phi 3) gaining adoption. Reasoning maturation: o1 and o3 models competitive on benchmarks, but still niche due to cost.

Parameter Growth Over Time

Key Insights from Timeline

  • Scaling Plateau: Bigger isn't always better. 7B models with good training match 13B models. Diminishing returns after 70B.
  • Efficiency Matters: MoE, quantization, and hardware-specific optimizations now as important as scale.
  • Open-Source Dominance: Llama ecosystem (7B-70B) drives more actual use than closed models.
  • Geographic Shift: Chinese models (Qwen) overtook US models in downloads (2024-2026).
  • Multimodal Default: By 2026, most new models include vision capabilities.
  • Specialized Beats General: Task-specific fine-tuning and tools beat larger general models in production.

Biggest Pain Points (Community Feedback)

1. Hallucinations

Models generate plausible-sounding but false information. Affects fact-heavy tasks (medicine, law).

Mitigation: RAG (grounding in documents), fact-checking steps, lower temperature.

2. Lost in the Middle

With long contexts (32K+), models perform worse on information in the middle. Prefer first/last chunks.

Mitigation: Reorder chunks by relevance, use smaller context windows, explicit retrieval.

3. Causality & Reasoning

Models struggle with causal relationships and multi-step logical reasoning (even large models).

Mitigation: Use reasoning models (o1, DeepSeek-R1), explicit chain-of-thought, human-in-the-loop.

4. Prompt Sensitivity

Small changes to prompts cause large output changes. Inconsistent behavior.

Mitigation: Structured prompts, few-shot examples, constraint-based generation.

When to Choose Commercial vs. Open-Source

✅ Use Commercial (1T+ models) When:

  • Maximum quality is critical (97%+ benchmarks)
  • Reasoning depth required (research, analysis)
  • Minimal hallucination acceptable (2-3%)
  • Zero-shot performance needed (no fine-tuning)
  • Multimodal required (Gemini 2.0)
  • Company budget >$100K/year for AI

✅ Use Open-Source (70B) When:

  • 90-92% quality acceptable (good enough)
  • Cost-sensitive (1-10% of commercial)
  • Privacy critical (run locally)
  • Custom fine-tuning needed (LoRA)
  • Latency critical (<100ms)
  • Control over deployment required

Hybrid Strategy: Commercial + Open-Source
