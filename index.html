Frequently Asked Questions

Q: Which model should I use for my project?
Start with the Model Selector tool. Answer 4-5 questions about your use case, and it will recommend the best model(s) with explanations.
Q: What's the difference between open-source and commercial models?
See the Complete Documentation Commercial Models section. Commercial models (1T+ params) offer 92-97% quality; open-source (70B) offers 90-92% at 1-10% cost.
Q: What is quantization and why does it matter?
Check the Technical Deep Dive → Quantization section. It explains FP32, INT8, GPTQ, and AWQ with 75% VRAM savings at only 5-10% quality loss.
Q: How do I fine-tune a model on my own data?
Head to Training & Fine-tuning. It has 3 levels: beginner (20 lines), intermediate (LoRA), advanced (QLoRA with 4-bit quantization).
Q: What parameters affect model output?
See Interactive Demos or Technical Deep Dive → Parameters. Learn about temperature (0-2), top-p, penalties, and context window effects.
Q: How do I choose between Llama, Qwen, Mistral, Claude, GPT-4?
The Magic Quadrant in Technical Deep Dive shows adoption vs maturity. Llama 3.3 70B is the production standard; Claude 3.5/Opus for best quality.
Q: How much VRAM do I need?
Complete Documentation has a hardware requirements table. Generally: 7B needs 4GB (4-bit), 70B needs 40GB (4-bit), commercial models need API access.
Q: What's the cost difference between models at scale?
Technical Deep Dive → Commercial Models has cost tables. Opus 4.6: $15+/1M tokens; Claude 3.5: $3-15/1M; Open-source: $0-2/1M or free locally.

📊 What's Covered in Each Guide?

1. Model Selector (model_selector.html)

🎯 Interactive tool | 5-10 min

Tasks (chat, code, embeddings, vision), priorities (quality/speed/cost/balance), constraints (VRAM, latency, multimodal), preferences (open-source, mobile). Algorithm scores 15+ models and ranks them.

2. Complete Documentation (models_documentation.html)

📚 Reference guide | 30-45 min read

6 categories: LLMs (open-source), commercial models (1T+ params), embeddings, vision, audio, multimodal, specialized. 50+ models with Hub links, benchmarks, comparison tables, selection guides, infrastructure requirements.

3. Technical Deep Dive (models_deep_dive.html)

🔬 Advanced learning | 1-2 hour read

Transformer architecture (attention, tokens, O(n²)), paradigms (encoder/decoder/encoder-decoder), parameters (temp, top-p, context), quantization (FP32→INT8→GPTQ), timeline (2017-2026), magic quadrant, community sentiment, commercial model benchmarks.

4. Interactive Demos (interactive_demos.html)

🎮 Hands-on | 20-30 min exploration

6 interactive sections: text generation (adjust temp/top-p/penalties), embeddings (similarity), classification (zero-shot), parameter tuning visualization, quantization comparison, training simulation (FT vs LoRA vs QLoRA).

5. Training & Fine-tuning (training_guide.html)

🚀 Implementation guide | 1-2 hour read + coding

3 difficulty levels: beginner (20-line HF Trainer), intermediate (LoRA r=8), advanced (QLoRA 4-bit). Data formats (JSONL, CSV, conversation), infrastructure (cloud/local), troubleshooting (OOM, convergence).
